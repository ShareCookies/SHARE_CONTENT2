Linux的发行版:
	发行版就是将Linux内核与应用软件做一个打包。
	一般来说linux系统基本上分两大类： 
		1.RedHat系列：Redhat、Centos、Fedora等 
		2.Debian系列：Debian、Ubuntu等 
Linux程序:

快捷键：
	tab键会自动补全文件名
字符含义: 
	~ :是home目录的意思
		以超级用户root账号登陆：~ 是 /root目录
		以普通用户name登陆：~是 /home/name目录
	/:
		根目录
		Linux最顶层只有一个根目录，所有文件都存储在这一个根目录之下。
	|:管道
		把前一个命令的结果当作，|后命令的输入
		例：
			ps -ef|grep java	
			ps -ef 查找所有程序的运行情况
			grep java 搜索给定的文件，得到与给定模式匹配的行
	?	
		//代表任意一个字符，如man？，那么man1就可以被其匹配到
	> 输出重定向：
		> 是定向输出到文件：
		如果文件不存在，就创建文件；
		如果文件存在，就将其清空；
		一般我们备份清理日志文件的时候，就是这种方法：先备份日志，再用`>`，将日志文件清空（文件大小变成0字节）；
	>> 这个是将输出内容追加到目标文件中：
		如果文件不存在，就创建文件；
		如果文件存在，则将新的内容追加到那个文件的末尾，该文件中的原有内容不受影响。
	--和-：
		两个后面一般都是跟参数选项
		--后面一般跟长的完整名字 如 chkconfig --add
		-后面一般跟简写大多数是一个字母，如 ls -l
帮助文档:
	man、info、help,cheat!!!
	man：man 命令//manual显示系统的手册页,也就是查看帮助文档，如果系统没有文档，是不会显示这些帮助信息的，比help要详细。它可以用来查看命令，函数，
 文件。帮助文档在/usr/share/man下。
	help:命令--help 选项，用来显示工具的一些信息，用法
	info:包括一些关于Linux Shell，工具，GNU项目开发程序说明文档,不但显示一些选项，还会有一些范例
	注:一般man用的比较多，比help细，比info简单
软件：	
	1.Linux软件安装方式:
		https://blog.csdn.net/aosica321/article/details/77085512
		http://blog.sina.com.cn/s/blog_55128b090102xn4h.html
		wget：
			类似于迅雷，是一种下载工具。
			通过HTTP、HTTPS、FTP三个最常见的TCP/IP协议下载。
		rpm：
			Red-Hat Package Manager（RPM软件包管理器）
			rpm 只能安装已经下载到本地机器上的rpm 包.
		yum：
			Yellow dog Updater, Modified（yum软件包管理器）
			基于RPM包管理，能够从指定的服务器自动下载RPM包并且安装，
			可以自动处理依赖性关系，并且一次安装所有依赖的软件包，无须繁琐地一次次下载、安装。
			例子：
				1.安装：
					yum install package1 安装指定的安装包package1
				2.更新：
					yum update package1 更新指定程序包package1
		dpkg：
			Debian Package是Debian系统的后台包管理器,类似RPM。
		apt：
			Advanced Packaging Tool（apt软件包管理工具）
			apt包管理工具类似yum,主要用于自动从互联网的软件仓库中搜索、安装、升级、卸载软件或操作系统。
		串联：
		   使用wget下载一个 rpm包, 然后用 rpm -ivh  xxx.rpm  安装这个软件
		   或可以直接用  yum  install  sqoop   来自动下载和安装依赖的rpm软件。
		以源码安装软件:
			先下载源码包文件：xxxx.tgz。
			解压源码包，然后根据情况来进行安装。
			例：
				1、解包软件
					tar zxf xxxx.tgz
				2、配置
					cd xxxx
					./configure ....
				3、编译
					make
				4、安装
					make install
				5、卸载        
					make uninstall		  
	2.linux下如何查看某软件是否已安装:
		https://www.cnblogs.com/yuanqiangfei/p/8033000.html
		因为linux安装软件的方式比较多，所以没有一个通用的办法能查到某些软件是否安装了。
		总结起来就是这样几类：
			1、rpm包安装的:
				可以用rpm -qa看到（查询系统中安装的所有RPM软件包），
				如果要查找某软件包是否安装，用 rpm -qa | grep 软件或包的名字。
			2、以deb包安装的：
				可以用dpkg -l能看到。
				如果是查找指定软件包，用dpkg -l | grep 软件或者包的名字。
			3、yum方法安装的：
				可以用yum list installed查找，
				如果是查找指定包，命令后加 | grep 软件名或者包名；
			4、如果是以源码包自己编译安装的，
				例如.tar.gz或者tar.bz2形式的，这个只能看可执行文件是否存在了，
				上面两种方法都看不到这种源码形式安装的包。
				如果是以root用户安装的，可执行程序通常都在/sbin:/usr/bin（/usr/local/bin）目录下。
				最可靠的去看NSTALL和README文件，一般都会有说明。
			注：
				其中rpm yum 是Redhat系linux的软件包管理命令，dpkg是debian系列的软件包管理命令
