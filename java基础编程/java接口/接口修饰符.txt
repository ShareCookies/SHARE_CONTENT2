接口访问修饰符：
	介绍：
		接口中每个方法都会被隐式指定为public abstract，所以接口每个方法都是隐式抽象的。
	接口中能用的访问修饰符：
		default（还是public级别，推荐使用）
			例： void test();
		public
		注：
			接口实现类方法修饰符，只能是public。
	不能用的访问修饰符：	
		private：抽象方法没有方法体，是用来被继承的，所以不能用private修饰；
		protected：protected访问修饰符也不能使用，因为接口可以让所有的类去实现（非继承），不只是其子类。
接口中无法使用的修饰符：
	static、synchronized、native等。
Attention：
	0.推荐：永远手动给出修饰符。
	1.修饰符
		1.0访问修饰符：
			可见修饰符：protected,private,public
			默认修饰符：package（package不是显示的修饰符，它是隐含的，即如果在类、变量等前没加显示的可见性修饰符，那它就是package级别的。所以用package修饰类会报错）
		1.1非访问修饰符：abstract、static、final。
			synchronized 
			transient
			volatile
